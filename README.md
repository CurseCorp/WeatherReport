# 🌦️ WeatherReport 

Асинхронное десктопное приложение для мониторинга погоды с возможностью 7-дневного прогноза. Проект разработан в рамках учебного курса по программной инженерии и системам управления.

В основе приложения лежит архитектурный паттерн **MVVM (Model-View-ViewModel)**, реализованный на базе C++ и фреймворка Qt/QML.

---

## 🛠️ Стек технологий

* **Язык программирования:** C++17 / C++20
* **Фреймворк:** Qt 6.x (Core, QML, Quick, Network)
* **Система сборки:** CMake
* **Архитектура:** MVVM (Model-View-ViewModel)
* **API данных:** Weather API
* **База данных (Кэш):** Postgres / QSettings

---

## 📐 Архитектура проекта (MVVM)

Приложение разделено на три независимых слоя, что позволило эффективно распределить задачи внутри команды разработки:

1.  **View (Слой отображения):** Написан на **QML**. Отвечает за отзывчивый UI, анимации, переключение экранов и ввод данных пользователем. Полностью отделен от бизнес-логики.
2.  **ViewModel (Прослойка):** Написана на **C++ (Qt)**. Выступает в роли диспетчера. Связывает View и Model с помощью механизма `Сигналов и Слотов (Qt Signals & Slots)` и `Q_PROPERTY`. Обрабатывает данные от Модели и адаптирует их для вывода на экран.
3.  **Model (Бизнес-логика):** Написана на **C++**. Включает в себя `WeatherService` и `ConfigService`. Отвечает за асинхронные сетевые запросы к OpenWeatherMap, парсинг JSON-ответов и кэширование данных.

## 🚀 Основной функционал

* **Асинхронный движок:** Интерфейс приложения остается плавным и отзывчивым во время выполнения сетевых запросов благодаря неблокирующей архитектуре `Qt ::connect`.
* **Детальный прогноз:** Текущая погода + прогноз изменений температуры на 7 дней вперед.
* **Гибкие настройки:** Окно конфигурации для динамического ввода и сохранения пользовательского API-ключа.
* **Локальное кэширование:** Сохранение избранных городов и настроек между сессиями (используется `QSettings`).
